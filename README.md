# Quick Capture

Native macOS app: global hotkey → tiny floating panel → type → ⌘-enter → file lands in `~/Documents/cxkb/raw/inbox/`. The missing low-friction *capture* end of the [cxkb](~/Documents/cxkb/) pipeline.

> **Status: seed.** Scaffolded 2026-05-21 from the cxkb seed. No code yet. Pick up here.

## The product is the spec

Design target: **<50ms from hotkey press to typing.** That is the entire product. If it's slower, the thought escapes and the app stops getting used.

### v0.1 (the only thing that has to exist)

- Global hotkey ⌘-shift-space (configurable later, hard-coded for v0.1)
- Borderless floating panel (NSPanel), no dock icon (`LSUIElement = YES`)
- Multi-line markdown text view
- ⌘-enter writes file to `~/Documents/cxkb/raw/inbox/YYYY-MM-DD-HHMM-<slug>.md` and dismisses
- ESC dismisses without saving
- Slug = autogenerated from first line (slugified) or "untitled"
- Frontmatter: `captured: <ISO timestamp>`, `hotkey: cmd-shift-space`

That's it. Nothing else in v0.1.

### Deferred to v0.2+

- Configurable hotkey
- Configurable target directory
- Optional auto-context (focused app, URL if browser)
- Voice capture variant (hold hotkey → Whisper)
- Auto-commit / git integration

## Decisions to make when you sit down

1. **Project structure** — Xcode `.xcodeproj` or SwiftPM `Package.swift`? Probably Xcode for a real menu bar app with `Info.plist` for `LSUIElement`. SwiftPM if you want to keep it dead-simple for now and worry about packaging later.
2. **Global hotkey API** — `Carbon.HIToolbox` (`RegisterEventHotKey`) is the established path. `MASShortcut` library wraps it. Skip the SwiftUI `KeyboardShortcut` API — that only works inside a focused app, not globally.
3. **Floating panel** — `NSPanel` with `.nonactivatingPanel` + `.canBecomeKey` styleMask. Must `setLevel(.floating)` and `becomeKeyOnlyIfNeeded = true` so it appears without stealing focus from the underlying app weirdly.
4. **Distribution** — for personal use, just build + drag to /Applications. Skip notarization until/unless you share it.

## Why native (not Electron/Tauri)

- <50ms latency is the spec. Electron *can* theoretically hit it; AppKit reliably does.
- Borderless floating panel that doesn't steal focus weirdly is easier in AppKit.
- LSUIElement background app is a native shape.
